# Story 1.1 — Modelos de Domínio + Roles + Auditoria

**Status:** Ready for Review
**Épico:** 1 — Parte 1

## User Story
Como **administrador do sistema**,
Quero **os modelos de domínio (Aluno, Professor, Disciplina, Sala, Aula, Presença), papéis de usuário (Admin/Professor/Aluno) e registro de auditoria**,
Para que **o sistema tenha a base de dados e o controle de acesso necessários para as demais funcionalidades da Parte 1**.

## Acceptance Criteria

- [x] AC1: App Django `core` criado contendo os modelos: `Aluno`, `Professor`, `Disciplina`, `Sala`, `Aula`, `Presenca` conforme campos do PDF (pág. 2-3).
- [x] AC2: `Sala` possui `latitude`, `longitude` e `raio_permitido` (default 50m).
- [x] AC3: `Aula` possui `token_qrcode` (UUID, único, gerado automaticamente ao criar).
- [x] AC4: `Presenca` possui `aluno`, `aula`, `horario_registro`, `ip_registrado`, `latitude`, `longitude`, `status`, e constraint `unique_together(aluno, aula)`.
- [x] AC5: **Roles via Groups nativos do Django** (`django.contrib.auth.Group`): `Administradores`, `Professores`, `Alunos`. Sem custom User, sem Profile — papel = Group do usuário. Admin = `is_staff=True`.
- [x] AC6: Groups criados via data migration com permissões apropriadas (usar `Permission` + `contenttypes` nativos).
- [x] AC7: **Auditoria via `django.contrib.admin.models.LogEntry`** para ações feitas no admin (já vem pronto). Para CRUD fora do admin, usar o mesmo `LogEntry` chamando `LogEntry.objects.log_action(...)` dentro das views (Story 1.2). Sem modelo novo.
- [x] AC8: Todos os modelos registrados no admin com `list_display`, `search_fields` e `list_filter` adequados (ModelAdmin padrão).
- [x] AC9: Migrations geradas e aplicadas com sucesso; `python manage.py check` sem warnings.
- [x] AC10: Senha segue políticas do Django (`AUTH_PASSWORD_VALIDATORS` já ativo) — nenhuma regressão.

## Notas Técnicas
- **Sem custom User nem Profile.** `User.groups` já resolve o papel. Helpers simples: `user.groups.filter(name='Professores').exists()`.
- `Aula.token_qrcode`: `UUIDField(default=uuid.uuid4, unique=True, editable=False)`.
- Ligação User↔Aluno/Professor: `OneToOneField(User)` em `Aluno` e `Professor` — necessário para identificar quem é quem (sem inventar Profile).
- IP institucional será validado na Story 1.4 — aqui só o campo em `Presenca`.
