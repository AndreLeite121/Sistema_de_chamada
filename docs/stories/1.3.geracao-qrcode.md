# Story 1.3 — Geração de QR Code por Aula

**Status:** Ready for Review
**Épico:** 1 — Parte 1
**Depende de:** 1.1, 1.2

## User Story
Como **professor**,
Quero **gerar um QR Code único para cada aula que eu crio**,
Para que **eu possa imprimi-lo e levá-lo à sala, permitindo que os alunos registrem presença ao escanear**.

## Acceptance Criteria

- [x] AC1: Dependência `qrcode[pil]` adicionada ao projeto.
- [x] AC2: View `aula_qrcode(request, aula_id)` que gera e retorna a imagem PNG do QR Code da aula.
- [x] AC3: Conteúdo do QR Code conforme PDF pág. 3: URL no formato `https://<host>/presenca?id=<aula_id>&token=<token>` usando `Aula.token_qrcode`.
- [x] AC4: Página "Imprimir QR" (`aula_imprimir.html`) exibindo: nome da disciplina, sala, data, horário, QR Code em alta resolução, layout amigável para impressão (CSS `@media print`).
- [x] AC5: Na listagem de Aulas (Story 1.2), botão "Gerar QR" visível apenas ao Professor responsável e ao Admin.
- [x] AC6: Endpoint de QR Code protegido — apenas Professor responsável pela aula e Admin acessam.
- [x] AC7: Regeneração do token disponível via ação explícita "Regenerar QR Code" (cria novo UUID, invalida o anterior).
- [x] AC8: Host base para a URL configurável via `settings.SITE_URL` (com default para dev: `http://localhost:8000`).

## Notas Técnicas
- Gerar imagem em memória com `io.BytesIO` e retornar `HttpResponse(content_type='image/png')`.
- Para impressão, embutir o QR como `<img src="{% url 'aula_qrcode' aula.id %}">`.

### Issues

**🟡 Issue-1 (MEDIUM) — `SITE_URL` hardcoded, não lê do env**
- **Local:** `cursodjango/settings.py:133`
- **Problema:** quando for para produção (Story 1.6 migra Postgres + `.env`), `SITE_URL` deveria vir de `os.environ.get('SITE_URL', 'http://localhost:8000')`.
- **Fix sugerido (leve, 1 linha):** `SITE_URL = os.environ.get('SITE_URL', 'http://localhost:8000')` + já usar `os`.
- **Impacto:** nenhum em dev; redeploy manual em prod.

**🟡 Issue-2 (MEDIUM) — Junção frágil da URL com `SITE_URL`**
- **Local:** `core/views.py:367` `f'{settings.SITE_URL}/presenca?...'`
- **Problema:** se `SITE_URL` vier com `/` no final, gera `//presenca`.
- **Fix sugerido:** `f"{settings.SITE_URL.rstrip('/')}/presenca?..."`.

**🟡 Issue-3 (MEDIUM) — Resolução do QR Code pode ficar baixa para impressão**
- **Local:** `qrcode.make(url)` usa `box_size=10` default (~300x300 px), template renderiza em `w-96 h-96` (~384px).
- **Problema:** em impressão A4, 384px pode pixelar ao esticar.
- **Fix sugerido:** `qrcode.QRCode(box_size=20, border=4)` ou gerar SVG (vetor, escala sem perda).
- **Impacto:** cosmético; QR ainda decodifica.

**🟡 Issue-4 (LOW) — Cache do navegador no PNG do QR após regeneração**
- **Local:** `aula_qrcode` retorna sem headers de cache.
- **Problema:** após `aula_regenerar_token`, browser pode servir PNG antigo do cache se URL não mudar.
- **Fix sugerido:** `response['Cache-Control'] = 'no-store'` OU versionar URL (`?v={token}`).
- **Workaround atual:** ctrl+F5 no preview.

**🟡 Issue-5 (LOW) — Error-correction level padrão**
- **Local:** `qrcode.make` default é `ERROR_CORRECT_M` (~15% de tolerância a sujeira).
- **Sugestão:** `ERROR_CORRECT_H` (30%) é mais robusto para QR físico em sala com má iluminação.

**🟢 Issue-6 (INFO) — Sem testes automatizados**
- Acompanha Issue-5 da Story 1.2 (já deferido). Quando criar test suite, cobrir:
  - `aula_qrcode` retorna 200 + `image/png` para dono
  - Non-owner retorna 403
  - `aula_regenerar_token` muda `token_qrcode` e grava `LogEntry`
