## User Story
Como **mantenedor do projeto**,
Quero **o banco de dados em PostgreSQL e as dependências fixadas em um `requirements.txt`**,
Para que **o sistema atenda ao requisito do roteiro (PDF pág. 1) e possa ser reproduzido em qualquer ambiente Linux**.

## Acceptance Criteria

- [x] AC1: `requirements.txt` criado com todas as dependências pinadas (Django, django-allauth, psycopg[binary], qrcode[pil], python-dotenv).
- [x] AC2: `settings.py` lê configs do banco via `.env`: `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT`.
- [x] AC3: Usa `django.db.backends.postgresql` com `psycopg` (v3).
- [x] AC4: `.env.example` criado com variáveis documentadas (sem valores reais).
- [x] AC5: `.gitignore` cobre: `venv/`, `__pycache__/`, `*.pyc`, `.env`, `db.sqlite3`, `staticfiles/`.
- [x] AC6: Script `scripts/setup_db.sh` (Linux) que cria database e user no Postgres local — opcional mas documentado.
- [x] AC7: `README.md` (novo ou atualizado) com passo a passo: clonar → venv → `pip install -r requirements.txt` → configurar `.env` → `createdb` → `migrate` → `runserver`.

- [x] AC9: `python manage.py check --deploy` sem erros críticos (warnings de prod aceitáveis para ambiente dev).

## Notas Técnicas
- `psycopg` v3 é o driver recomendado em Django 6.
- Manter compatibilidade com SQLite via flag `DB_ENGINE` (opcional): facilita dev.

### Issues

**🟡 Issue-1 (MEDIUM) — SECRET_KEY insegura como default**
- **Local:** `settings.py` L21-24 — fallback mantém chave `django-insecure-...`
- **Impacto:** se operador subir prod sem `.env`, roda com chave insegura e conhecida.
- **Fix sugerido:** em prod (`DEBUG=False`), `raise ImproperlyConfigured` se `SECRET_KEY` env não foi setado.

**🟢 Issue-2 (LOW) — Migrate contra Postgres não testado neste review**
- **Reason:** ambiente do QA sem Postgres.
- **Ação do usuário:** rodar localmente `createdb sistema_freq && DB_ENGINE=postgresql DB_PASSWORD=xyz python manage.py migrate` e confirmar que as 2 migrations (`0001_initial` + `0002_groups`) aplicam limpas.

**🟢 Issue-3 (INFO) — Fallback SQLite pode mascarar problemas**
- Se dev esquecer de setar `.env`, roda em SQLite mesmo intenção sendo Postgres. Nota didática no README poderia alertar.
